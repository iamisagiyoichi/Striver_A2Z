# Lec 1: Things to Know in C++ / Java / Python
# Topic: Data Types

A **Data Type** defines the **type of data** a variable can store, the **amount of memory** it occupies, the **range of values** it can hold, and the **operations** that can be performed on it.

Without data types, the compiler/interpreter would not know how much memory to allocate or how to interpret the stored value.

---

# Why Do We Need Data Types?

Suppose you want to store:

- Age → Integer
- Salary → Decimal Number
- Grade → Character
- Name → String
- Is Student? → True/False

Each of these requires a different type of storage.

Example:

```java
int age = 20;
double salary = 85000.50;
char grade = 'A';
String name = "Anuj";
boolean isStudent = true;
```

# Primitive Data Types

Primitive data types store **actual values**.

| Data Type | Size | Range |
|------------|------|--------------------------------|
| byte | 1 Byte | -128 to 127 |
| short | 2 Bytes | -32,768 to 32,767 |
| int | 4 Bytes | -2³¹ to 2³¹-1 |
| long | 8 Bytes | -2⁶³ to 2⁶³-1 |
| float | 4 Bytes | ~6-7 decimal digits |
| double | 8 Bytes | ~15-16 decimal digits |
| char | 2 Bytes | 0 to 65,535 (Unicode) |
| boolean | JVM Dependent | true / false |

---

# 1. byte

Stores very small integers.

```java
byte x = 100;
```

Memory

```
1 Byte = 8 bits
```

Range

```
-128 to 127
```

Use Case

- Saving memory
- Large arrays with small values

Range

Rarely used in Competitive Programming.

# 3. int

Most commonly used integer type.

```java

```

Size

```
4 Bytes
```

Range

```
-2,147,483,648

to
```

---

# 4. long

Stores very large integers.

```java
long population = 8000000000L;
```

Notice the suffix

```java
L
```

Without `L`

```java
long x = 10000000000;
```

Compilation Error.

---

### When Should You Use `long`?

Use `long` whenever:

```
Constraints ≥ 10⁹
```

or

Multiplication can overflow.

Example

Wrong

```java
int area = 100000 * 100000;
```

Overflow occurs.

Correct

```java
long area = 100000L * 100000L;
```

Unlike primitive types, these store a **reference (memory address)** to the actual object.

Examples

```java

int[] arr = new int[5];

ArrayList<Integer> list = new ArrayList<>();
```

Memory Representation

```
Stack

name ----->

Heap

"Anuj"
```

# Type Casting

Type casting converts one data type into another.

---

## 1. Implicit Casting (Widening)

Smaller → Larger

```java
int x = 10;

long y = x;
```

No data loss.

---

## 2. Explicit Casting (Narrowing)

Larger → Smaller

```java
double d = 10.75;

int x = (int)d;
```

Output

```
10
```

Decimal part is discarded.

---

# Integer Overflow

One of the most common bugs in DSA.

Example

```java
int x = 100000;

int y = 100000;

System.out.println(x * y);
```

Expected

```
10000000000
```

Actual

Overflow occurs because the result exceeds the `int` range.

Correct

```java
long ans = 1L * x * y;
```

# Data Types in Competitive Programming

### Use `int` when

- Constraints ≤ 10⁹
- No multiplication overflow

---

### Use `long` when

- Constraints > 10⁹
- Prefix sums
- Large answers
- Products of numbers

---

### Use `double` when

- Precision is required
- Geometry problems
- Probability

---

### Use `char`

- Character frequency
- String traversal
- ASCII manipulation

---

# Common Mistakes

### Forgetting `L`

Wrong

```java
long x = 10000000000;
```

Correct

```java
long x = 10000000000L;
```

---

### Forgetting `f`

Wrong

```java
float x = 3.14;
```

Correct

```java
float x = 3.14f;
```

---

### Character in Double Quotes

Wrong

```java
char c = "A";
```

Correct

```java
char c = 'A';
```

---

### Integer Overflow

Wrong

```java
int ans = a * b;
```

Correct

```java
long ans = 1L * a * b;
```

---

### Comparing Decimal Numbers

Avoid direct comparison due to precision errors.

Wrong

```java
if(a == b)
```

Better

```java
if(Math.abs(a - b) < 1e-9)
```

---

# Revision Notes

- Data types define memory, range, and operations.
- Primitive types store actual values.
- Reference types store object references.
- `int` is the default integer type.
- `long` is safer for large numbers and multiplications.
- `double` is preferred over `float` for decimal values.
- `char` stores a single Unicode character.
- `boolean` stores only `true` or `false`.
- Use implicit casting for widening and explicit casting for narrowing.
- Always watch out for integer overflow.
