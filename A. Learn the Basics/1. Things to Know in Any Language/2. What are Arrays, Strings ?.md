# Topic: Arrays and Strings

Arrays and Strings are two of the most fundamental data structures in programming. Almost every Data Structures & Algorithms (DSA) problem is built upon these concepts.

---

# 1. What is an Array?

An **Array** is a linear data structure that stores multiple elements of the **same data type** in **contiguous memory locations**.

Think of an array as a row of lockers. Each locker has a unique number (index), and each locker stores one value.

```
Array

Index

0   1   2   3   4

+---+---+---+---+---+
|10 |20 |30 |40 |50 |
+---+---+---+---+---+
```

---

# Why Do We Need Arrays?

Without arrays:

```java
int a = 10;
int b = 20;
int c = 30;
int d = 40;
```

With arrays:

```java
int[] arr = {10, 20, 30, 40};
```

Instead of creating multiple variables, we can store all related values together.

---

# Characteristics of Arrays

- Stores elements of the same data type.
- Elements are stored in contiguous memory.
- Indexing starts from **0**.
- Fixed size after creation (in Java).
- Supports random access in constant time.

---

---

# Accessing Elements

```java
System.out.println(arr[2]);
```

Output

```
2
```

Access Time

```
O(1)
```

Why?

Because the address of any element can be calculated directly:

```
Address = Base Address + (Index × Size of Data Type)
```

---

# Updating Elements

```java

```


# Time Complexity of Array Operations

| Operation | Complexity |
|-----------|------------|
| Access | O(1) |
| Update | O(1) |
| Traversal | O(N) |
| Search (Unsorted) | O(N) |
| Search (Sorted - Binary Search) | O(log N) |
| Insert at End | O(1)* |
| Insert/Delete at Beginning or Middle | O(N) |

\*Only if space is available.

---

# Advantages of Arrays

- Fast random access.
- Easy to traverse.
- Cache-friendly due to contiguous memory.
- Low memory overhead.

---

# Disadvantages of Arrays

- Fixed size.
- Insertion and deletion are expensive.
- Stores only one data type.
- May waste memory if allocated larger than needed.

# 2. What is a String?

A **String** is a sequence of characters used to represent text.

Example

```java
String name = "Anuj";
```

Memory

```
Index

0 1 2 3

A n u j
```

---

# Why Strings?

Strings allow us to store and manipulate textual data.

Examples

- Names
- Emails
- Passwords
- URLs
- Sentences

---

# Characteristics of Strings

- Collection of characters.
- Indexed from **0**.
- Immutable in Java.
- Supports many built-in methods.

---

# String Immutability

Strings in Java cannot be modified after creation.

```java
String s = "Hello";

s.concat(" World");

System.out.println(s);
```

Output

```
Hello
```

Because `concat()` creates a **new String** instead of modifying the existing one.

Correct

```java
s = s.concat(" World");
```

# StringBuilder

Since Strings are immutable, repeated modifications create many new objects.

Use `StringBuilder` for efficient modifications.

```java
StringBuilder sb = new StringBuilder();
```

Appending

```java
sb.append("Hello");
```

Reverse

```java
sb.reverse();
```

Insert

```java
sb.insert(2, 'A');
```

Delete

```java
sb.delete(2,5);
```

Convert Back

```java
String str = sb.toString();
```

# Common Mistakes

### Array Index Out of Bounds

Wrong

```java
arr[arr.length]
```

Correct

```java
arr[arr.length - 1]
```

---

### Comparing Strings

Wrong

```java
if(s1 == s2)
```

Correct

```java
if(s1.equals(s2))
```

---

### Modifying Strings

Wrong

```java
s.concat("abc");
```

Correct

```java
s = s.concat("abc");
```

or

```java
StringBuilder sb = new StringBuilder(s);
sb.append("abc");
```

# Revision Notes

### Arrays

- Stores same data type.
- Contiguous memory.
- Fixed size.
- O(1) indexing.
- Insertion/Deletion is O(N).
- Great for random access.

### Strings

- Sequence of characters.
- Immutable in Java.
- O(1) character access.
- Use `equals()` for comparison.
- Use `StringBuilder` for frequent modifications.
